# Анализ GitHub-репозиториев из исследовательских документов

**Источники**:
- `Поиск примеров скомкивания бумаги в Blender.pdf`
- `Примеры 3D-симуляции сжатия бумаги (скомкивания).pdf`

Ниже приведен анализ ссылок, извлеченных из документов, с приоритезацией для текущей задачи (симуляция развертывания скомканной бумаги).

## Сводная таблица

| Приоритет | Репозиторий | Описание / Контекст | Обоснование |
| :--- | :--- | :--- | :--- |
| **HIGH** | [doc3D-renderer](https://github.com/sagniklp/doc3D-renderer) | Рендеринг 3D-документов с деформациями. Используется для создания датасетов по выравниванию (dewarping). | Специализированный код для документов. |
| **HIGH** | [UVDoc](https://github.com/tanguymagne/UVDoc) | Модель и данные для выравнивания документов. | Фокус на геометрии бумаги и UV-картах. |
| **MEDIUM** | [cloth-funnels](https://github.com/real-stanford/cloth-funnels) | Роботизированные манипуляции с тканью (Stanford). | **Upgraded per feedback**: Использует Blender Cloth Sim. При настройке высокой жесткости (Bending Stiffness) и трения является отличным референсом для *динамики* сминания. Содержит готовые скрипты для генерации случайных состояний. |
| **MEDIUM** | [synthetic-cloth-data](https://github.com/tlpss/synthetic-cloth-data) | Генерация синтетических данных ткани. | Полезен для изучения параметров Cloth Modifier, которые делают материал похожим на жесткую ткань/бумагу. |
| **MEDIUM** | [flingbot](https://github.com/real-stanford/flingbot) | Динамическое разворачивание ткани. | Алгоритмы развертывания (unfolding strategies) могут быть адаптированы для бумаги, если учитывать ограничение на растяжение. |
| **MEDIUM** | [DewarpNet](https://github.com/cvlab-stonybrook/DewarpNet) | Классическая работа по выравниванию сканов. | "Золотой стандарт" для статических датасетов. |
| **LOW** | [Augraphy](https://github.com/sparkfish/augraphy) | 2D-аугментация (шум, пятна, сгибы). | Только 2D. |

## План исследования (Action Plan)

1.  **Сравнение с Cloth Sim (cloth-funnels)**:
    *   Изучить, какие параметры `structural_stiffness` и `bending_stiffness` они используют.
    *   Если они используют стандартный Blender Cloth, это подтверждает валидность нашего направления (Experiment 13), но, возможно, у них есть трюки с "Pinning" или "Self Collision", которые мы упустили.

2.  **Генерация складок (doc3D)**:
    *   По-прежнему актуально для *инициализации* меша.

3.  **Гибридный подход**:
    *   Использовать настройки материалов из `cloth-funnels` + алгоритм сминания из `doc3D`.
